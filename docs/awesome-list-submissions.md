# Awesome List Submission Targets

This file tracks outreach targets for making AgentShield easier to find from
MCP, Claude, OpenAI Agents, LangGraph, CrewAI, and browser automation discovery
paths.

## Suggested Listing Blurb

```markdown
- [AgentShield](https://github.com/aiconnai/agentshield) - Offline Rust security
  scanner for MCP and AI agent extensions, with multi-framework adapters and
  SARIF output for GitHub Code Scanning.
```

## Targets

| Target | Why it fits | Status |
|--------|-------------|--------|
| `e2b-dev/awesome-ai-sdks` | Accepts SDKs, frameworks, libraries, and tools for creating, monitoring, debugging, and deploying autonomous AI agents. | Submitted PR: https://github.com/e2b-dev/awesome-ai-sdks/pull/240 |
| `appcypher/awesome-mcp-servers` | Has a `Tools & Utilities` area and the README tells users to review MCP server code before install. | Prepared branch: https://github.com/limaronaldo/awesome-mcp-servers/tree/add-agentshield. PR creation is currently blocked by GitHub permissions for the authenticated token, and upstream issues are disabled. |
| `punkpeye/awesome-mcp-servers` | Large MCP discovery list. | Not submitted: the list focuses on MCP server implementations, and recent AgentShield-named submissions appear unrelated to this scanner. Avoiding another submission unless maintainers explicitly accept standalone security tooling. |
| Claude MCP community lists | Claude Desktop and Claude Code users install MCP servers from community lists. | Submit only under security, tooling, or developer utilities. |
| OpenAI Agents, LangGraph, and CrewAI resource lists | AgentShield protects agent tools across those ecosystems. | Submit only where third-party security tooling is accepted. |
