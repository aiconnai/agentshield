# AgentShield 0.8.6 Release Notes

Release date: 2026-06-07

## Scope

This release ships the AGENT-22 bidirectional MCP proxy transport behind the
`runtime-guard` feature. It builds on the 0.8.5 MCP proxy decision core without
changing AgentShield's stable static scanner output contracts.

## Notable changes

- Added `agentshield guard --mcp-proxy -- <server cmd...>`, which spawns a
  downstream MCP server and bridges line-delimited stdio in both directions.
- Allowed and warning `tools/call` requests are forwarded as original JSON-RPC
  bytes, and the downstream server's responses are streamed back to the host.
- Blocked calls are answered by the proxy with safe JSON-RPC error `-32001`;
  the downstream MCP server never receives those requests.
- Downstream write failures now return JSON-RPC error `-32002`
  (`Downstream MCP server unavailable`) instead of silently dropping allowed
  traffic.
- The transport pump locks stdout per response line to avoid deadlocks while
  still keeping response writes coherent.

## Compatibility

- `agentshield guard --mcp-proxy` without a server command keeps the 0.8.5
  line-protocol mode and emits `{"forward": <request>}` for allowed traffic.
- Scanner console, JSON, SARIF, HTML, and DSSE output contracts are unchanged.
- The MCP proxy guard remains experimental pending broader MCP client/server
  compatibility testing and stable runtime compatibility guarantees.
