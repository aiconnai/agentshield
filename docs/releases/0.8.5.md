# AgentShield 0.8.5 Release Notes

Release date: 2026-06-07

## Scope

This release ships the experimental AGENT-20 MCP proxy guard surface behind the
`runtime-guard` feature while preserving AgentShield's stable static scanner
contracts.

## Notable changes

- Added `agentshield guard --mcp-proxy`, a line-delimited JSON-RPC guard loop
  for MCP traffic. Non-`tools/call` messages pass through. `tools/call` messages
  are evaluated with the shared runtime guard core before forwarding.
- Added `[runtime.proxy]` config with `fail_on = "block" | "warn" | "never"`
  and per-tool `[[runtime.proxy.tool]]` overrides.
- Blocked calls return a safe JSON-RPC error using code `-32001` and only expose
  verdict, rule ID, and schema version.
- Hardened the proxy against nested or string-typed metadata-service SSRF
  arguments, oversized input lines, and silent `never` suppression.

## Readiness

- The MCP proxy guard remains experimental.
- The 0.8.5 implementation provides the policy decision core and stdio
  line-protocol surface. A full bidirectional transport that spawns or connects
  to a downstream MCP server remains future production hardening work.
- Scanner console, JSON, SARIF, and HTML output contracts are unchanged.
