# AgentShield 0.8.3 Release Notes

Release date: 2026-06-05

## Scope

AgentShield 0.8.3 is a release hygiene update for the 0.8 line. It keeps the 0.8 feature scope intact while fixing release-version drift, modernizing GitHub Actions runtime usage, and publishing a full-feature Docker image for both common Linux architectures.

## Notable changes since 0.8.0

- Bumped the crate version to `0.8.3` so release binaries report the same version as the published tag.
- Added a release workflow guard that fails if `v<version>` does not match `Cargo.toml`.
- Updated direct GitHub Actions usage for Node 24 readiness and enabled `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24`.
- Built release binaries with `--features full`, including runtime `wrap` support.
- Built the GHCR image with `--features full`, so Docker users get `agentshield wrap`.
- Published the GHCR image for `linux/amd64` and `linux/arm64`.

## Published artifacts

- GitHub release: `v0.8.3`
- Docker image: `ghcr.io/aiconnai/agentshield:0.8.3`
- Docker platforms: `linux/amd64`, `linux/arm64`

## Verification notes

- Release workflow passed for Linux, macOS, and Windows binary artifacts.
- Docker/GHCR workflow passed and published the multi-arch image.
- The GHCR image reports `agentshield 0.8.3`.
- The GHCR image exposes the `wrap` command.

## Known operational note

The GHCR package visibility is controlled by GitHub package settings. If the package remains private, authenticated pulls require a GitHub token with package read access.
